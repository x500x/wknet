本账本覆盖 M7 协议路径的对端可控输入和所有 `ProtocolAllocationSite`。每个条目必须同时具备 owner、上限、计数、溢出检查、错误映射、释放点和测试；普通 HTTP buffered response 不使用低位全局容量兜底，`MaxResponseBytes=0` 仍表示调用方未设置上限。

## 故障注入覆盖

`ProtocolAllocationSite` 编号 1–58（`Count=59`）固定在 `rtl/ProtocolFailureInjection.h`。用户态测试对每个实际生产 site 设置 fail-on-next/fail-on-nth，并在失败后检查对应 live counter；`protocol_failure_injection_tests` 另外验证编号枚举、fail-on-nth 语义、fail-always 和释放下溢保护。最终门禁运行 `Protocol`、`AllFirstParty` 和显式 `quic_stress_tests`，不能只枚举 site 而不经过 owner 的真实分配路径。
