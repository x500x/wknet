# 证书信任库偏好

- PEM `CertificateAuthorityBundle` 是独立 CA 证书成员的集合，不能因为一个无关成员无法解析或暂不支持，就阻断后续有效信任锚的搜索。
- 被隔离的成员不得参与信任判断；跳过不可用成员不是宽松证书校验，也不能降低目标证书链的主机名、时间、用途、路径、签名、信任锚或撤销检查要求。
- 只有在当前 PEM 块的 `BEGIN`/`END` 边界完整、能够可靠定位下一成员时，才允许隔离该成员的格式或能力错误。
- 缺失 PEM 结束边界、无法可靠继续扫描、内存分配失败、参数错误或密码学执行错误必须原样失败，禁止吞掉。
- 单 DER AuthorityBundle 没有独立成员恢复边界，解析失败必须严格返回，不能按 PEM 集合语义跳过。
- 外部信任包成员被隔离时应记录证书序号、偏移和 NTSTATUS，便于审计具体不兼容证书。
- 证书信任库回归测试必须包含真实离线 CA bundle 的完整扫描，确保无关旧证书或暂不支持证书不会污染其他信任锚。
